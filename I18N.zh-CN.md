# I18N 多语言规范

**中文** | [English](./I18N.md)

## 1）范围

本仓库采用双语维护：

1. 英文（`en`）作为默认主文档（`*.md`）
2. 简体中文（`zh-CN`）作为镜像文档（`*.zh-CN.md`）

## 2）命名规范

1. 主文档：`*.md`（英文）
2. 中文镜像：`*.zh-CN.md`
3. 仓库入口：`README.md`（英文优先，顶部提供语言切换）
4. 重要文档尽量保持中英成对。

## 3）同步规则

1. 先更新英文源文档。
2. 在同一个 PR 或后续 PR 同步中文镜像。
3. 翻译 PR 需标注源文档路径与源 commit/PR。
4. 保持章节编号、公式、字段语义一致。
5. 对高风险段落（模型数据、公式、规范约束）优先保证语义一致，不追求花哨表达。

## 4）多语言适配参考项目

1. Kubernetes Website：<https://github.com/kubernetes/website>
2. Ant Design：<https://github.com/ant-design/ant-design>
3. Element Plus：<https://github.com/element-plus/element-plus>
4. Docusaurus：<https://github.com/facebook/docusaurus>
5. pnpm Website：<https://github.com/pnpm/pnpm.io>
6. Antdv Pro：<https://github.com/antdv-pro/antdv-pro>

这些项目的可复用模式包括：

1. README 语言入口
2. 中英镜像文件
3. 翻译同步流程（人工或 Crowdin）
4. 文档目录分层

## 5）本仓库当前落地策略

当前采用轻量方案：

1. 使用 `.md + .zh-CN.md` 的成对文件
2. 暂不强制迁移为 `docs/en + docs/zh-CN`
3. 等规模和贡献者数量上来后再升级目录形态

## 6）评审检查清单

1. 有镜像时，链接是否优先跳转同语言文档？
2. 日期、数字、模型名、公式是否一致？
3. 脱敏是否在双语中都保持一致？
4. 是否存在未翻译段落导致规范语义偏移？

## 7）语言属性闸门（必过）

1. 每个 `*.md` 文件必须是英文主文内容，不能变成中英混写或中文主文。
2. 每个重要 `*.md` 文件都应有配套 `*.zh-CN.md` 镜像（若临时缺失，需在 PR 描述中明确说明原因）。
3. 若 PR 改坏了源文档语言属性（例如误覆盖），必须在同一变更中修复。
4. 闸门不通过则不合并，先修复语言归属与镜像一致性。
5. 每个新增 `*.md` 是否保持英文主文档，并同步了对应 `*.zh-CN.md` 镜像？
