# 贡献指南
**中文** | [English](./CONTRIBUTING.md)

感谢你参与 Robust LLM-Wiki。

本项目是“实践框架仓库”，目标是沉淀方法、规范和可复用流程，而不是沉淀个人内容。

## 1. 贡献范围

欢迎贡献：

1. `research/`：可复核的数据、调研结论、案例归纳
2. `schema/`：规范、红线、执行建议
3. 跨文档一致性与工作流治理口径

不建议贡献：

1. 与项目目标无关的个人内容页
2. 未经说明的大规模风格重写
3. 未经讨论的脚本化“强约束模板”

## 2. 贡献顺序（重要）

默认顺序：

1. 先改 `schema/details/*`
2. 再同步 `schema/SPEC.md`
3. 最后做术语与边界一致性检查

## 3. 提交前自检

1. 是否保持 Karpathy 内核：Wiki、双链、ingest/query/lint 闭环
2. 是否使用通俗表达，先讲“为什么”，再讲“怎么做”
3. 是否完成强脱敏（不得暴露真实姓名、昵称、账号、站点与可反查身份线索；统一使用“实体-A/B”代号）
4. 是否补充来源与口径说明（尤其是模型评测数据）
5. 是否避免把建议写成“一刀切硬规则”

## 4. Pull Request 建议结构

请在 PR 描述里说明：

1. 改动动机（Why）
2. 主要改动文件
3. 影响范围（哪些规范/流程被影响）
4. 仍未覆盖的风险或待办

## 5. 讨论与提案

对于较大改动（新增规范章节、调整总纲结构、重写关键定义），建议先提 Issue/讨论，再提交 PR。
