# Open Source Policy (Framework)
**中文** | [English](./OPEN_SOURCE_POLICY.md)

本项目定位为开源框架项目，默认遵循以下开源协议与治理约束。

## 1. License 策略

1. 代码与脚本：建议采用 `Apache-2.0`（包含专利授权条款，适合框架项目）。
2. 研究与规范文档：建议采用 `CC BY 4.0`（允许引用与再分发，保留署名）。
3. 所有文件建议附带 SPDX 标识，便于自动化扫描。

## 2. 贡献协议

1. 贡献者提交 PR 时应确认其有权提交代码与文档。
2. 建议启用 DCO（Developer Certificate of Origin）签署流程。
3. 第三方代码、素材、文档必须明确来源与许可证。

## 3. 合规与审计

1. 禁止提交无法确认许可来源的代码片段。
2. 引入依赖时，需记录许可证类型与版本。
3. 保留依赖与第三方引用清单，便于后续审计。

## 4. 社区协作规范

1. 讨论、Issue、PR 行为遵循尊重与建设性原则。
2. 与安全相关的问题应走私下披露流程，而非公开细节。

## 5. 对本项目的落地建议

1. 若需要“最少约束、快速扩散”，可切换为 MIT。
2. 若需要“框架级治理和专利安全”，推荐维持 Apache-2.0。
3. 无论采用哪种许可证，都应在 README 中明确并保持一致。

## 6. 项目协作文件映射

1. 贡献流程：`CONTRIBUTING.md`
