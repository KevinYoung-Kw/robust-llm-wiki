# Open Source Policy (Framework)
[中文](./OPEN_SOURCE_POLICY.zh-CN.md) | **English**

This project is positioned as an open-source framework project and, by default, follows the open-source policy and governance constraints below.

## 1. License Strategy

1. Code and scripts: recommend `Apache-2.0` (includes patent grant clauses and is suitable for framework projects).
2. Research and specification documents: recommend `CC BY 4.0` (allows citation and redistribution with attribution).
3. It is recommended to include SPDX identifiers in all files for automated scanning.

## 2. Contribution Agreement

1. Contributors should confirm they have the rights to submit code and documentation when opening PRs.
2. Enabling a DCO (Developer Certificate of Origin) signing workflow is recommended.
3. Third-party code, assets, and documents must include explicit source and license information.

## 3. Compliance And Audit

1. Do not submit code snippets whose license source cannot be verified.
2. When introducing dependencies, record both license type and version.
3. Keep dependency and third-party reference inventories for future audits.

## 4. Community Collaboration Norms

1. Discussions, Issues, and PR behavior should follow respectful and constructive principles.
2. Security-related issues should follow a private disclosure process instead of public detail disclosure.

## 5. Practical Recommendations For This Project

1. If you need “minimum constraints and rapid diffusion,” you may switch to MIT.
2. If you need “framework-level governance and patent safety,” Apache-2.0 is recommended.
3. Regardless of license choice, state it clearly in README and keep it consistent.
